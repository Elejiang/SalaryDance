# 风险点与检查清单

本文档列出项目高风险修改点和代码审查清单；开发前检查、排查回归或做 review 时加载。

## 高风险区域

| 区域 | 风险 | 检查点 |
|------|------|--------|
| `SalaryViewModel` | 跨夜工作窗口 | 晚班、下班后、昨日窗口仍在进行中 |
| `SalaryConfig` | 配置默认值和计算 | 现行配置结构、默认值、normalize |
| `OffTaskTracker` | 摸鱼区间归属和自动结算 | 跨夜、休息不计薪、下班后未手动结束 |

## 禁止事项

- 构建产物、DMG、本地 Xcode 用户状态不进入 Git。
- 配置模型以现行结构为准，不维护历史字段迁移。
- 工作进度和计薪进度是两个概念；休息不计薪时，工作时间百分比仍继续走。
- 计薪周期和月薪计薪方式是两个独立配置；固定 21.75 只影响月薪折算分母，不能改变本周期/本月范围。
- 按月补贴的“平摊到每天”只影响日薪和实时收入；月薪/年薪汇总仍按按月补贴原值计入。
- 关闭的补贴必须完整排除出日薪、月薪、年薪、实时收入和节假日数据预加载判断。
- 摸鱼薪资必须按有效计薪区间计算，不能把不计薪休息段或下班后的打开状态算入金额。
- 摸鱼本周、本周期/本月统计必须复用 `OffTaskTracker` 的区间汇总，不能绕过有效计薪区间或用户配置的 `SalaryCycleMode`。
- 摸鱼历史编辑只能改时间范围；已结束记录改开始和结束时间，进行中记录改开始时间，不能在记录编辑入口直接改金额、时长或其他结算结果。
- 摸鱼历史删除必须删除整条原始记录，并触发当日、周期、累计和年-月-日层级统计同步刷新。

## 新增配置字段清单

新增 `SalaryConfig` 字段时检查：

1. 字段有默认值。
2. `normalize()` 修正非法值。
3. 如果影响展示，设置页和预览同步更新。
4. 如果影响计算，`SPEC.md` 或 `config-and-calculation.md` 同步更新。
5. 字段加入 `CodingKeys` 和自定义 `init(from:)`，用 `decodeLossy` / `decodeIfPresent` 兼容旧配置。
6. 新增或改名枚举展示文案时，不修改持久化 raw value；历史文案别名放到枚举自定义解码里。

## 新增 Swift 文件清单

新增 Swift 文件时检查：

1. 文件放在合适目录。
2. 加入 `SalaryDance.xcodeproj/project.pbxproj` 的 FileReference、Group、Sources。
3. 构建通过。
4. 文档索引需要时同步更新。
