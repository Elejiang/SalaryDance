# 风险点与检查清单

本文档列出项目高风险修改点和代码审查清单；开发前检查、排查回归或做 review 时加载。

## 高风险区域

| 区域 | 风险 | 检查点 |
|------|------|--------|
| `SalaryViewModel` | 跨夜工作窗口 | 晚班、下班后、昨日窗口仍在进行中 |
| `SalaryConfig` | 配置默认值和计算 | 现行配置结构、默认值、normalize |

## 禁止事项

- 构建产物、DMG、本地 Xcode 用户状态不进入 Git。
- 配置模型以现行结构为准，不维护历史字段迁移。
- 工作进度和计薪进度是两个概念；休息不计薪时，工作时间百分比仍继续走。
- 月薪折算由配置模式决定，固定 21.75 仅适用于固定天数模式。
- 按月补贴的“平摊到每天”只影响日薪和实时收入；月薪/年薪汇总仍按按月补贴原值计入。
- 关闭的补贴必须完整排除出日薪、月薪、年薪、实时收入和节假日数据预加载判断。

## 新增配置字段清单

新增 `SalaryConfig` 字段时检查：

1. 字段有默认值。
2. `normalize()` 修正非法值。
3. 如果影响展示，设置页和预览同步更新。
4. 如果影响计算，`SPEC.md` 或 `config-and-calculation.md` 同步更新。
5. 配置不兼容时重置本地配置。

## 新增 Swift 文件清单

新增 Swift 文件时检查：

1. 文件放在合适目录。
2. 加入 `SalaryDance.xcodeproj/project.pbxproj` 的 FileReference、Group、Sources。
3. 构建通过。
4. 文档索引需要时同步更新。
