# 配置、薪资计算与刷新机制

本文档说明配置字段、薪资换算、工作时间、节假日和刷新机制；修改 `SalaryConfig`、`SalaryViewModel` 或节假日逻辑时加载。

## 配置存储

| 项 | 说明 |
|----|------|
| 存储 | `UserDefaults.standard` |
| 主配置 key | `salary_config` |
| 编解码 | `SalaryConfig` 使用 `JSONEncoder` / `JSONDecoder` |
| 节假日 key | `chinese_holidays_data_by_year` |
| 节假日结构 | `[CachedHolidays]` |

策略：

- 配置模型以现行结构为准，不维护历史字段迁移。
- 配置不兼容时可以重置本地配置。

## 默认配置

默认值集中在 `SalaryConfig`：

| 配置 | 默认值 |
|------|--------|
| 薪资类型 | 月薪 |
| 金额 | 0 |
| 补贴 | 空列表 |
| 新增补贴 | 默认开启，名称为“补贴名” |
| 按月补贴平摊固定天数 | 21.75 |
| 工作时间 | 10:00 - 21:00 |
| 午休 | 开启，12:00 - 14:00 |
| 晚饭 | 开启，18:00 - 19:00 |
| 计薪规则 | 仅工作日 |
| 休息时间计薪 | 关闭 |
| 快捷键 | Option + Command + Z |
| 刷新间隔 | 1 秒，范围 0.5-3600 |
| 空闲低频刷新 | 开启 |

## 薪资换算

用户输入的基础薪资先统一折算为基础日薪：

| 输入类型 | 基础日薪 |
|----------|------|
| 日薪 | 输入值 |
| 月薪 | 月薪 / 月薪折算天数 |
| 年薪 | 年薪 / 250 |

补贴再按自身规则计入：

关闭的补贴保留配置但不参与任何计算，方便用户临时对比开启前后的实时收入和换算结果。

| 补贴类型 | 对日薪的影响 | 对月薪/年薪的影响 |
|----------|--------------|-------------------|
| 按日补贴 | 直接加到展示日薪，进入实时收入 | 月薪按 `按日补贴 × 月薪折算天数`，年薪按 `按日补贴 × 250` |
| 按月补贴 + 加到月薪 | 不进入展示日薪和实时收入 | 月薪按月度原值计入，年薪按 `月度原值 × 12` |
| 按月补贴 + 平摊到每天 | 按平摊分母折成日补贴，进入实时收入 | 月薪仍按月度原值计入，年薪按 `月度原值 × 12` |

最终展示值：

| 展示值 | 公式 |
|--------|------|
| 日薪 | 基础日薪 + 按日补贴 + 平摊到每天的按月补贴 |
| 月薪 | 基础日薪 × 月薪折算天数 + 按日补贴 × 月薪折算天数 + 按月补贴原值 |
| 年薪 | （基础日薪 + 按日补贴）× 250 + 按月补贴原值 × 12 |
| 时薪 | 秒薪 × 3600 |
| 分薪 | 秒薪 × 60 |
| 秒薪 | 展示日薪 / 当前计薪秒数 |

## 月薪折算

`MonthlySalaryCalculationMode` 有两种：

| 模式 | 说明 |
|------|------|
| 固定指定天数 | 默认 21.75，范围 1-31 |
| 按薪资周期计薪日 | 根据当前薪资周期内计薪日数量动态计算 |

动态周期注意：

- 起始日支持 1-31。
- 当月没有对应日期时，按当月最后一天作为周期起点。
- 计薪天数按“日历”里的计薪规则统计。
- 不同周期工作日数量不同，输入日薪时更适合使用该方式。

## 补贴平摊

按月补贴选择“平摊到每天”时，`MonthlySubsidyProrationMode` 有三种：

| 模式 | 说明 |
|------|------|
| 周期内总天数 | 使用当前薪资周期自然日总数作为分母 |
| 固定天数 | 默认 21.75，范围 1-31 |
| 周期内工作日天数 | 使用当前薪资周期内按计薪规则统计出的工作日数量 |

注意：

- 平摊只决定是否进入展示日薪、实时收入、秒薪、分薪和时薪。
- 月薪汇总始终按该按月补贴的月度原值计入，避免因为平摊分母和月薪折算天数不同而丢失补贴金额。
- 周期内工作日天数依赖节假日/调休日数据；仅工作日模式下需要确保当前薪资周期涉及年份已加载。

## 工作时间与休息

- `TimeRange.durationMinutes` 支持结束时间小于开始时间，表示跨夜。
- `SalaryViewModel` 会先判断昨天开始的工作窗口是否仍在进行中，再判断今天窗口。
- 午休和晚饭会裁剪到工作窗口内。
- 午休和晚饭可能重叠，扣减计薪时长前必须合并，避免重复扣。
- `breakTimeCountsAsPaidWork = true` 时，休息时间计入计薪时长。

## 工作进度

- 工作进度百分比按完整工作窗口变化。
- 即使休息时间不计薪，百分比仍然继续走。
- 小数位由 `workProgressDecimalPlaces` 控制，范围 0-3。
- 真实弹窗和设置预览都要使用同一格式。

## 节假日

`ChineseHolidays` 负责自动获取和缓存节假日/调休日：

- 调休日优先级高于周末。
- 法定节假日覆盖普通工作日。
- 年初/年末会加载相邻年份，覆盖跨年薪资周期。

## 刷新机制

`SalaryViewModel` 使用 `Timer`：

| 场景 | 间隔 |
|------|------|
| 弹窗打开 | 用户配置间隔 |
| 状态栏实时金额开启 | 用户配置间隔 |
| 无实时展示且空闲低频开启 | `max(60, resolvedRefreshIntervalSeconds)` |
| 无实时展示且空闲低频关闭 | 用户配置间隔 |

Timer tolerance：

```swift
min(5, max(0.2, interval * 0.25))
```

即时刷新触发点：

- 配置变更。
- 快捷键打开或关闭状态栏实时显示。
- 弹窗打开前。
