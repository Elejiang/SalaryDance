# 配置、薪资计算与刷新机制

本文档说明配置字段、薪资换算、工作时间、节假日和刷新机制；修改 `SalaryConfig`、`SalaryViewModel` 或节假日逻辑时加载。

## 配置存储

| 项 | 说明 |
|----|------|
| 存储 | `UserDefaults.standard` |
| 主配置 key | `salary_config` |
| 编解码 | `SalaryConfig` 使用 `JSONEncoder` / `JSONDecoder` |
| 节假日 key | `chinese_holidays_data_by_year` |
| 节假日结构 | `[CachedHolidays]` |

策略：

- 配置模型以现行结构为准，不维护历史字段迁移。
- 配置不兼容时可以重置本地配置。

## 默认配置

默认值集中在 `SalaryConfig`：

| 配置 | 默认值 |
|------|--------|
| 薪资类型 | 月薪 |
| 金额 | 0 |
| 工作时间 | 10:00 - 21:00 |
| 午休 | 开启，12:00 - 14:00 |
| 晚饭 | 开启，18:00 - 19:00 |
| 计薪规则 | 仅工作日 |
| 休息时间计薪 | 关闭 |
| 快捷键 | Option + Command + Z |
| 刷新间隔 | 1 秒，范围 0.5-3600 |
| 空闲低频刷新 | 开启 |

## 薪资换算

所有输入统一先折算为日薪：

| 输入类型 | 日薪 |
|----------|------|
| 日薪 | 输入值 |
| 月薪 | 月薪 / 月薪折算天数 |
| 年薪 | 年薪 / 250 |

再从日薪派生：

| 展示值 | 公式 |
|--------|------|
| 月薪 | 日薪 × 月薪折算天数 |
| 年薪 | 日薪 × 250 |
| 时薪 | 秒薪 × 3600 |
| 分薪 | 秒薪 × 60 |
| 秒薪 | 日薪 / 当前计薪秒数 |

## 月薪折算

`MonthlySalaryCalculationMode` 有两种：

| 模式 | 说明 |
|------|------|
| 固定指定天数 | 默认 21.75，范围 1-31 |
| 按薪资周期计薪日 | 根据当前薪资周期内计薪日数量动态计算 |

动态周期注意：

- 起始日支持 1-31。
- 当月没有对应日期时，按当月最后一天作为周期起点。
- 计薪天数按“日历”里的计薪规则统计。
- 不同周期工作日数量不同，输入日薪时更适合使用该方式。

## 工作时间与休息

- `TimeRange.durationMinutes` 支持结束时间小于开始时间，表示跨夜。
- `SalaryViewModel` 会先判断昨天开始的工作窗口是否仍在进行中，再判断今天窗口。
- 午休和晚饭会裁剪到工作窗口内。
- 午休和晚饭可能重叠，扣减计薪时长前必须合并，避免重复扣。
- `breakTimeCountsAsPaidWork = true` 时，休息时间计入计薪时长。

## 工作进度

- 工作进度百分比按完整工作窗口变化。
- 即使休息时间不计薪，百分比仍然继续走。
- 小数位由 `workProgressDecimalPlaces` 控制，范围 0-3。
- 真实弹窗和设置预览都要使用同一格式。

## 节假日

`ChineseHolidays` 负责自动获取和缓存节假日/调休日：

- 调休日优先级高于周末。
- 法定节假日覆盖普通工作日。
- 年初/年末会加载相邻年份，覆盖跨年薪资周期。

## 刷新机制

`SalaryViewModel` 使用 `Timer`：

| 场景 | 间隔 |
|------|------|
| 弹窗打开 | 用户配置间隔 |
| 状态栏实时金额开启 | 用户配置间隔 |
| 无实时展示且空闲低频开启 | `max(60, resolvedRefreshIntervalSeconds)` |
| 无实时展示且空闲低频关闭 | 用户配置间隔 |

Timer tolerance：

```swift
min(5, max(0.2, interval * 0.25))
```

即时刷新触发点：

- 配置变更。
- 快捷键打开或关闭状态栏实时显示。
- 弹窗打开前。
