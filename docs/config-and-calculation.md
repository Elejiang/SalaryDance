# 配置、薪资计算与刷新机制

职责：维护持久化、导入导出、薪资换算、工作时间、节假日和刷新机制规则。

## 持久化与导入导出

| 项 | 说明 |
|----|------|
| 存储 | `UserDefaults.standard` |
| 主配置 key | `salary_config`，由 `SalaryConfigManager` 读写 |
| 主配置结构 | `SalaryConfig`，包含薪资数据和偏好配置 |
| 摸鱼记录 key | `off_task_sessions` |
| 摸鱼记录结构 | `[OffTaskSession]` |
| 提前下班记录 key | `clock_out_sessions` |
| 提前下班记录结构 | `[ClockOutSession]` |
| 晚下班记录 key | `overtime_sessions` |
| 晚下班记录结构 | `[OvertimeSession]` |
| 节假日 key | `chinese_holidays_data_by_year` |
| 节假日结构 | `[CachedHolidays]` |
| 数据导入导出 | JSON 文件，包含薪资、补贴、计薪规则、工作时间、`[OffTaskSession]` 摸鱼记录、`[ClockOutSession]` 提前下班记录和 `[OvertimeSession]` 晚下班记录，不包含节假日缓存 |
| 配置导入导出 | JSON 文件，包含展示、快捷键、颜色、刷新、开机启动和设置侧边栏宽度等偏好配置 |

策略：

- `SalaryConfig` 和嵌套配置模型需要兼容旧字段；新增字段必须在自定义解码里提供默认值。
- 配置枚举的持久化值使用稳定 key，展示文案通过 `title` 提供；解码层兼容历史中文值。
- `salary_config` 解码彻底失败时，先备份到 `salary_config_decode_failed_backup`，错误信息写入 `salary_config_decode_failed_error`，再使用内存默认配置启动，避免静默覆盖原始数据。
- 数据导入和配置导入字段零交集；两者合并覆盖除节假日缓存以外的可迁移内容。
- 导入前必须先完整解码并 normalize；数据导入还要校验摸鱼、提前下班和晚下班记录时间范围，确认后再替换当前分区。

## 薪资换算

用户输入的基础薪资先统一折算为基础日薪：

| 输入类型 | 基础日薪 |
|----------|------|
| 日薪 | 输入值 |
| 月薪 | 月薪 / 月薪折算天数 |
| 年薪 | 年薪 / 250 |

补贴再按自身规则计入：

关闭的补贴保留配置但不参与任何计算，方便用户对比开启前后的实时收入和换算结果。

| 补贴类型 | 对日薪的影响 | 对月薪/年薪的影响 |
|----------|--------------|-------------------|
| 按日补贴 | 直接加到展示日薪，进入实时收入 | 月薪按 `按日补贴 × 月薪折算天数`，年薪按 `按日补贴 × 250` |
| 按月补贴 + 加到月薪 | 不进入展示日薪和实时收入 | 月薪按月度原值计入，年薪按 `月度原值 × 12` |
| 按月补贴 + 平摊到每天 | 按平摊分母折成日补贴，进入实时收入 | 月薪仍按月度原值计入，年薪按 `月度原值 × 12` |

最终展示值：

| 展示值 | 公式 |
|--------|------|
| 日薪 | 基础日薪 + 按日补贴 + 平摊到每天的按月补贴 |
| 月薪 | 基础日薪 × 月薪折算天数 + 按日补贴 × 月薪折算天数 + 按月补贴原值 |
| 年薪 | （基础日薪 + 按日补贴）× 250 + 按月补贴原值 × 12 |
| 时薪 | 秒薪 × 3600 |
| 分薪 | 秒薪 × 60 |
| 秒薪 | 展示日薪 / 当天有效计薪秒数 |

## 月薪折算

月薪配置拆成两个互不替代的概念：`SalaryCycleMode` 决定金额、补贴和摸鱼统计归属到哪个日期范围，`MonthlySalaryCalculationMode` 只决定月薪与日薪互相折算时使用的分母。

计薪周期：

| 模式 | 说明 |
|------|------|
| 自然月 | 每月 1 日到当月最后一天 |
| 固定周期 | 每月指定起始日到次月前一日，起始日支持 1-31 |

月薪计薪方式：

| 模式 | 说明 |
|------|------|
| 固定天数 | 默认 21.75，范围 1-31 |
| 周期内工作天数 | 使用当前计薪周期内按“日历”规则统计出的计薪日数量 |

周期注意：

- 固定周期的当月没有对应日期时，按当月最后一天作为周期起点。
- 切换固定天数/周期内工作天数不会改变当前计薪周期范围。
- 摸鱼本周期/本月统计必须跟随 `SalaryCycleMode`；自然月展示为“本月”，固定周期展示为“本周期”。
- 周期内工作天数依赖“日历”里的计薪规则；不同周期工作日数量不同，输入日薪时更适合使用该方式。

## 补贴平摊

按月补贴选择“平摊到每天”时，`MonthlySubsidyProrationMode` 有三种：

| 模式 | 说明 |
|------|------|
| 周期内总天数 | 使用当前计薪周期自然日总数作为分母 |
| 固定天数 | 默认 21.75，范围 1-31 |
| 周期内工作日天数 | 使用当前计薪周期内按计薪规则统计出的工作日数量 |

注意：

- 平摊只决定是否进入展示日薪、实时收入、秒薪、分薪和时薪。
- 月薪汇总始终按该按月补贴的月度原值计入，避免因为平摊分母和月薪折算天数不同而丢失补贴金额。
- 周期内工作日天数依赖节假日/调休日数据；仅工作日模式下需要确保当前计薪周期涉及年份已加载。

## 工作时间与休息

- `TimeRange.durationMinutes` 支持结束时间小于开始时间，表示跨夜。
- `SalaryViewModel` 会先判断昨天开始的工作窗口是否仍在进行中，再判断今天窗口。
- 午休和晚饭会裁剪到工作窗口内。
- 午休和晚饭可能重叠，扣减计薪时长前必须合并，避免重复扣。
- `breakTimeCountsAsPaidWork = true` 时，休息时间计入计薪时长。

## 特殊工作日

`specialWorkdayRules` 是有序数组，数组顺序就是优先级：

- 规则可单独启用/关闭，关闭后保留配置但不参与匹配。
- 同一天命中多条规则时，只使用第一条启用且命中的规则。
- 特殊规则只覆盖当天上下班时间，不改变计薪日判断、休息时间、补贴、月薪或年薪折算。
- 命中特殊规则后，今日收入、工作进度、秒薪、分薪和时薪使用当天有效工作时间；日薪展示仍是当天日薪金额，月薪、年薪仍使用原有配置。

规则类型：

| 类型 | 说明 |
|------|------|
| 节假日和周末的前一天 | 今天计薪且明天按当前计薪规则不计薪时命中；仅工作日模式下会受节假日/调休日影响 |
| 固定星期 | 每周指定星期命中 |
| 隔周循环 | 从起始日期所在周开始，每 N 周的指定星期命中，N 范围 1-12 |
| 指定日期 | 指定自然日命中 |

## 工作进度

- 工作进度百分比按完整工作窗口变化。
- 即使休息时间不计薪，百分比仍然继续走。
- 小数位由 `workProgressDecimalPlaces` 控制，范围 0-3。
- 真实弹窗和设置预览都要使用同一格式。

## 摸鱼统计

- `OffTaskTracker` 使用 `off_task_sessions` 持久化摸鱼区间，`end == nil` 表示当前仍在摸鱼中。
- 摸鱼状态只允许在真实工作窗口内开启；如果用户一直开启到下班后，刷新时自动把打开区间结算到该工作窗口的下班时刻。
- `SalaryWorkTimeline` 统一展开工作窗口和有效计薪区间，摸鱼薪资按“摸鱼区间 ∩ 有效计薪区间”的秒数乘以当天秒薪；手动摸鱼记录必须落在同一个真实工作窗口内，结束时间可以等于该工作窗口的下班时间。
- 休息时间不计薪时，摸鱼统计同样扣除午休和晚饭；休息时间计薪时，休息段也会被计入摸鱼薪资。已有记录不会因为切换休息计薪规则而被删除，只会按当前规则重新结算：完全落在不计薪休息段的记录保留在历史中，但不计入摸鱼薪资、时长和次数。
- 摸鱼只能在有效计薪区间开启；午休/晚饭未计薪时休息期间不可开启，进行中的摸鱼记录跨入休息段时自动结算到休息开始。
- 跨夜班次按工作窗口起始日归属统计，凌晨仍在昨天班次内的摸鱼记录计入昨天。
- 每条记录可回溯编辑时间范围，精确到秒；已结束记录编辑开始和结束时间，进行中记录编辑开始时间，金额和计薪时长按当前薪资配置实时重算。
- 每条记录可删除，删除后对应时长、金额和次数立即从统计中移除。
- 摸鱼记录支持手动新增已结束区间，新增时结束时间不能超过当前时间。
- 历史摸鱼记录可按年、月、日分组一键删除，删除后对应维度内全部原始记录会从统计中移除。
- 历史摸鱼记录可一键按工作日合并同一天内时间重叠的记录；合并只改变原始分段，不改变当前薪资配置的实时换算规则。
- 设置页数据概览展示本日、本周、当前计薪周期三种维度；本周和当前计薪周期必须写明起止日期。
- 数据概览和历史记录中的摸鱼时长可展示占比，分子为对应范围内的摸鱼计薪时长；数据概览的本日、本周和当前计薪周期分母使用该范围内全部工作日的排班计薪时长，特殊工作日按当天覆盖后的上班时间计算。
- 历史累计摸鱼时长占比的分母从第一条摸鱼记录所在日期开始，统计到当前时刻已发生的计薪工作时长；历史列表按年、月、日折叠时，第一条记录所在年/月的分母起点使用第一条记录日期，后续年/月使用自然年、自然月起点并统计到当前已发生的计薪工作时长。
- 当前计薪周期沿用用户配置的 `SalaryCycleMode`，自然月展示为“本月”，固定周期展示为“本周期”；该范围不受月薪计薪方式影响。
- 历史记录展示历史累计和按年-月-日折叠的全部记录。
- 弹窗摸鱼状态内可单独展示今日摸鱼摘要，以及本日、本周、本月/本周期、历史的摸鱼薪资和摸鱼时长；展示开关由 `SalaryConfig` 的 `popoverShows...OffTask...` 字段控制。

## 提前下班与晚下班统计

- `WorkSessionTracker` 使用 `clock_out_sessions` 持久化提前下班记录，使用 `overtime_sessions` 持久化晚下班记录，内部结构名沿用 `OvertimeSession` 以兼容历史数据。
- 即时提前下班入口只允许在真实工作窗口内触发；触发后记录从点击时刻到当天真实下班时刻的区间，可撤回当天记录。
- 手动新增提前下班记录只输入实际下班时间，实际下班时间必须在真实工作窗口内且不超过当前时间；记录结束时间必须由该工作日的真实下班时间推算，特殊工作日使用其覆盖后的下班时间。
- 提前下班统计按“提前下班区间 ∩ 有效计薪区间”的秒数乘以当天秒薪，金额语义是提前下班仍赚到的薪资，不扣减今日收入。
- 一个工作日最多只能存在一条提前下班或晚下班记录；同一天已提前下班时不能再记录晚下班，同一天已晚下班时不能再记录提前下班。
- 即时晚下班入口只允许在真实下班后触发；用户输入或点击调整晚下班小时和分钟后生成一条计划区间，分钟点击调整步幅为 5 分钟，可提前终止或撤回当天晚下班记录。
- 手动新增晚下班记录只输入晚下班工作日和晚下班时长，记录开始时间必须由该工作日的真实下班时间推算；当天晚下班允许在下班前提前预定，非当天记录的晚下班结束时间不能超过当前时间，任何晚下班都不能进入下一次真实上班窗口。例如周四晚下班不能进入周五上班窗口，周五晚下班不能进入下一个计薪工作日的上班窗口。
- 同一工作日再次保存更长的晚下班时长时，不新建第二条记录，而是延长该工作日已有晚下班记录；延长后的结束时间仍必须早于下一次真实上班窗口。
- 晚下班默认无额外收入；统计按已发生的晚下班秒数乘以当天秒薪，金额语义是晚下班亏损，不追加今日收入。
- 晚下班记录处于进行中时，统计只累计到当前时刻；用户可提前终止当前晚下班，记录会以终止时刻作为结束时间进入结算。
- 跨夜班次按工作窗口起始日归属提前下班和晚下班记录。
- 提前下班和晚下班记录支持手动新增、单条编辑和单条删除，金额和时长仍按当前薪资配置实时重算；编辑时使用与新增相同的同日互斥、晚下班延长和下一次上班窗口校验规则。
- 提前下班和晚下班历史记录可按年、月、日分组一键删除，删除后对应维度内全部原始记录会从统计中移除。
- 设置页记录栏目展示本日、本周、当前计薪周期、历史累计和按年-月-日折叠的全部记录；本周和当前计薪周期必须写明起止日期。
- 弹窗提前下班与晚下班面板默认只保留提前下班/晚下班入口、晚下班时分选择和今日提示文案；不展示金额指标网格。
- 弹窗提前下班与晚下班的状态入口、今日提示、提前下班入口和晚下班入口可在展示设置中单独开关；状态入口默认关闭。
- 数据导入导出包含提前下班和晚下班原始记录，配置导入导出不包含这些行为记录。

## 节假日

`ChineseHolidays` 负责自动获取和缓存节假日/调休日：

- 调休日优先级高于周末。
- 法定节假日覆盖普通工作日。
- 年初/年末会加载相邻年份，覆盖跨年计薪周期。

## 刷新机制

`SalaryViewModel` 使用 `Timer`：

| 场景 | 间隔 |
|------|------|
| 弹窗打开 | 用户配置间隔 |
| 状态栏实时金额开启 | 用户配置间隔 |
| 无实时展示且空闲低频开启 | `max(60, resolvedRefreshIntervalSeconds)` |
| 无实时展示且空闲低频关闭 | 用户配置间隔 |

Timer tolerance：

| 间隔 | tolerance |
|------|-----------|
| `<= 1` 秒 | `0.02` |
| `<= 2` 秒 | `0.05` |
| `> 2` 秒 | `min(5, max(0.2, interval * 0.25))` |

即时刷新触发点：

- 配置变更。
- 快捷键打开或关闭状态栏实时显示。
- 快捷键开启或结束摸鱼状态。
- 弹窗提前下班、撤回提前下班、开始晚下班或撤回晚下班。
- 弹窗打开前。
