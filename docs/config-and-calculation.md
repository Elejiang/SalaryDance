# 配置、薪资计算与刷新机制

本文档说明持久化、导入导出、薪资换算、工作时间、节假日和刷新机制；修改 `SalaryConfig`、`SalaryViewModel`、`OffTaskTracker` 或节假日逻辑时加载。

## 持久化与导入导出

| 项 | 说明 |
|----|------|
| 存储 | `UserDefaults.standard` |
| 主配置 key | `salary_config`，由 `SalaryConfigManager` 读写 |
| 主配置结构 | `SalaryConfig`，包含薪资数据和偏好配置 |
| 摸鱼记录 key | `off_task_sessions` |
| 摸鱼记录结构 | `[OffTaskSession]` |
| 节假日 key | `chinese_holidays_data_by_year` |
| 节假日结构 | `[CachedHolidays]` |
| 数据导入导出 | JSON 文件，包含薪资、补贴、计薪规则、工作时间和 `[OffTaskSession]` 摸鱼记录，不包含节假日缓存 |
| 配置导入导出 | JSON 文件，包含展示、快捷键、颜色、刷新、开机启动等偏好配置 |

策略：

- `SalaryConfig` 和嵌套配置模型需要兼容旧字段；新增字段必须在自定义解码里提供默认值。
- 配置枚举的持久化值使用稳定 key，展示文案通过 `title` 提供；解码层兼容历史中文值。
- `salary_config` 解码彻底失败时，先备份到 `salary_config_decode_failed_backup`，错误信息写入 `salary_config_decode_failed_error`，再使用内存默认配置启动，避免静默覆盖原始数据。
- 数据导入和配置导入字段零交集；两者合并覆盖除节假日缓存以外的可迁移内容。
- 导入前必须先完整解码并 normalize；数据导入还要校验摸鱼记录时间范围和进行中记录数量，确认后再替换当前分区。

## 薪资换算

用户输入的基础薪资先统一折算为基础日薪：

| 输入类型 | 基础日薪 |
|----------|------|
| 日薪 | 输入值 |
| 月薪 | 月薪 / 月薪折算天数 |
| 年薪 | 年薪 / 250 |

补贴再按自身规则计入：

关闭的补贴保留配置但不参与任何计算，方便用户对比开启前后的实时收入和换算结果。

| 补贴类型 | 对日薪的影响 | 对月薪/年薪的影响 |
|----------|--------------|-------------------|
| 按日补贴 | 直接加到展示日薪，进入实时收入 | 月薪按 `按日补贴 × 月薪折算天数`，年薪按 `按日补贴 × 250` |
| 按月补贴 + 加到月薪 | 不进入展示日薪和实时收入 | 月薪按月度原值计入，年薪按 `月度原值 × 12` |
| 按月补贴 + 平摊到每天 | 按平摊分母折成日补贴，进入实时收入 | 月薪仍按月度原值计入，年薪按 `月度原值 × 12` |

最终展示值：

| 展示值 | 公式 |
|--------|------|
| 日薪 | 基础日薪 + 按日补贴 + 平摊到每天的按月补贴 |
| 月薪 | 基础日薪 × 月薪折算天数 + 按日补贴 × 月薪折算天数 + 按月补贴原值 |
| 年薪 | （基础日薪 + 按日补贴）× 250 + 按月补贴原值 × 12 |
| 时薪 | 秒薪 × 3600 |
| 分薪 | 秒薪 × 60 |
| 秒薪 | 展示日薪 / 当天有效计薪秒数 |

## 月薪折算

月薪配置拆成两个互不替代的概念：`SalaryCycleMode` 决定金额、补贴和摸鱼统计归属到哪个日期范围，`MonthlySalaryCalculationMode` 只决定月薪与日薪互相折算时使用的分母。

计薪周期：

| 模式 | 说明 |
|------|------|
| 自然月 | 每月 1 日到当月最后一天 |
| 固定周期 | 每月指定起始日到次月前一日，起始日支持 1-31 |

月薪计薪方式：

| 模式 | 说明 |
|------|------|
| 固定天数 | 默认 21.75，范围 1-31 |
| 周期内工作天数 | 使用当前计薪周期内按“日历”规则统计出的计薪日数量 |

周期注意：

- 固定周期的当月没有对应日期时，按当月最后一天作为周期起点。
- 切换固定天数/周期内工作天数不会改变当前计薪周期范围。
- 摸鱼本周期/本月统计必须跟随 `SalaryCycleMode`；自然月展示为“本月”，固定周期展示为“本周期”。
- 周期内工作天数依赖“日历”里的计薪规则；不同周期工作日数量不同，输入日薪时更适合使用该方式。

## 补贴平摊

按月补贴选择“平摊到每天”时，`MonthlySubsidyProrationMode` 有三种：

| 模式 | 说明 |
|------|------|
| 周期内总天数 | 使用当前计薪周期自然日总数作为分母 |
| 固定天数 | 默认 21.75，范围 1-31 |
| 周期内工作日天数 | 使用当前计薪周期内按计薪规则统计出的工作日数量 |

注意：

- 平摊只决定是否进入展示日薪、实时收入、秒薪、分薪和时薪。
- 月薪汇总始终按该按月补贴的月度原值计入，避免因为平摊分母和月薪折算天数不同而丢失补贴金额。
- 周期内工作日天数依赖节假日/调休日数据；仅工作日模式下需要确保当前计薪周期涉及年份已加载。

## 工作时间与休息

- `TimeRange.durationMinutes` 支持结束时间小于开始时间，表示跨夜。
- `SalaryViewModel` 会先判断昨天开始的工作窗口是否仍在进行中，再判断今天窗口。
- 午休和晚饭会裁剪到工作窗口内。
- 午休和晚饭可能重叠，扣减计薪时长前必须合并，避免重复扣。
- `breakTimeCountsAsPaidWork = true` 时，休息时间计入计薪时长。

## 特殊工作日

`specialWorkdayRules` 是有序数组，数组顺序就是优先级：

- 规则可单独启用/关闭，关闭后保留配置但不参与匹配。
- 同一天命中多条规则时，只使用第一条启用且命中的规则。
- 特殊规则只覆盖当天上下班时间，不改变计薪日判断、休息时间、补贴、月薪或年薪折算。
- 命中特殊规则后，今日收入、工作进度、秒薪、分薪和时薪使用当天有效工作时间；日薪展示仍是当天日薪金额，月薪、年薪仍使用原有配置。

规则类型：

| 类型 | 说明 |
|------|------|
| 节假日和周末的前一天 | 今天计薪且明天按当前计薪规则不计薪时命中；仅工作日模式下会受节假日/调休日影响 |
| 固定星期 | 每周指定星期命中 |
| 隔周循环 | 从起始日期所在周开始，每 N 周的指定星期命中，N 范围 1-12 |
| 指定日期 | 指定自然日命中 |

## 工作进度

- 工作进度百分比按完整工作窗口变化。
- 即使休息时间不计薪，百分比仍然继续走。
- 小数位由 `workProgressDecimalPlaces` 控制，范围 0-3。
- 真实弹窗和设置预览都要使用同一格式。

## 摸鱼统计

- `OffTaskTracker` 使用 `off_task_sessions` 持久化摸鱼区间，`end == nil` 表示当前仍在摸鱼中。
- 摸鱼状态只允许在真实工作窗口内开启；如果用户一直开启到下班后，刷新时自动把打开区间结算到该工作窗口的下班时刻。
- `SalaryWorkTimeline` 统一展开工作窗口和有效计薪区间，摸鱼薪资按“摸鱼区间 ∩ 有效计薪区间”的秒数乘以当天秒薪。
- 休息时间不计薪时，摸鱼统计同样扣除午休和晚饭；休息时间计薪时，休息段也会被计入摸鱼薪资。
- 摸鱼只能在有效计薪区间开启；午休/晚饭未计薪时休息期间不可开启，进行中的摸鱼记录跨入休息段时自动结算到休息开始。
- 跨夜班次按工作窗口起始日归属统计，凌晨仍在昨天班次内的摸鱼记录计入昨天。
- 每条记录可回溯编辑时间范围，精确到秒；已结束记录编辑开始和结束时间，进行中记录编辑开始时间，金额和计薪时长按当前薪资配置实时重算。
- 每条记录可删除，删除后对应时长、金额和次数立即从统计中移除。
- 设置页数据概览展示本日、本周、当前计薪周期三种维度；本周和当前计薪周期必须写明起止日期。
- 当前计薪周期沿用用户配置的 `SalaryCycleMode`，自然月展示为“本月”，固定周期展示为“本周期”；该范围不受月薪计薪方式影响。
- 历史记录展示历史累计和按年-月-日折叠的全部记录。
- 弹窗摸鱼状态内可单独展示本日、本周、本月/本周期、历史的摸鱼薪资和摸鱼时长；这些指标默认关闭。

## 节假日

`ChineseHolidays` 负责自动获取和缓存节假日/调休日：

- 调休日优先级高于周末。
- 法定节假日覆盖普通工作日。
- 年初/年末会加载相邻年份，覆盖跨年计薪周期。

## 刷新机制

`SalaryViewModel` 使用 `Timer`：

| 场景 | 间隔 |
|------|------|
| 弹窗打开 | 用户配置间隔 |
| 状态栏实时金额开启 | 用户配置间隔 |
| 无实时展示且空闲低频开启 | `max(60, resolvedRefreshIntervalSeconds)` |
| 无实时展示且空闲低频关闭 | 用户配置间隔 |

Timer tolerance：

```swift
min(5, max(0.2, interval * 0.25))
```

即时刷新触发点：

- 配置变更。
- 快捷键打开或关闭状态栏实时显示。
- 快捷键开启或结束摸鱼状态。
- 弹窗打开前。
